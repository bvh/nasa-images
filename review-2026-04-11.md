Overall the codebase is clean, well-organized, stdlib-only, and the architecture matches CLAUDE.md. Findings grouped by severity.

## Bugs

**~~1. DONE [__main__.py:95-100](nasa_images/__main__.py#L95-L100) — `call_asset` checks truthiness of the object, not `.okay`.~~**
~~Every other `call_*` uses `if captions.okay:` etc. `if manifest:` here relies on `Asset` being truthy, which it always is (no `__bool__`). Inconsistent with siblings and the falsy branch is unreachable. Should be `if manifest.okay:`.~~

**2. [__main__.py:59](nasa_images/__main__.py#L59) — `search` passes `args.id` as `nasa_id`.**
`--id` and `--album` are in a mutually exclusive group. CLAUDE.md's search signature lists `[--nasa-id TEXT]` but argparse only exposes `-i/--id`. Either add `--nasa-id` or fix the docs.

**~~3. DONE [fetch.py:117-122](nasa_images/fetch.py#L117-L122) — index save only happens on new fetches, not for every item.~~**
~~`_download_file` raising an exception propagates all the way up and aborts the album walk with a traceback rather than logging and continuing. Consider catching per-item so one bad asset doesn't kill a 3000-item album download.~~

**4. [fetch.py:71](nasa_images/fetch.py#L71) — `catalog_index.add(nasa_id)` happens even on partial failure.**
If the `~orig` file already exists (line 66 warning path) and metadata was missing, we still mark the ID as cataloged. A partially-downloaded asset that was interrupted in a prior run and left no `.part` will be skipped forever on re-run.

**~~5. DONE [inventory.py:19](inventory.py#L19) — `any(p.is_dir() for p in path.iterdir())` on an empty directory returns False, so empty dirs get listed as nasa_ids.~~** ~~Minor; an `unknown/` dir with nothing in it would show up as `"unknown"`.~~

## Code quality

~~**6. DONE [api.py:60-115](nasa_images/api.py#L60-L115) — massive duplication across `Asset`,~~
~~`Metadata`, `Album`, `Captions`.** All four have identical 7-line constructors differing only in the endpoint URL. A single base `__init__` taking a path segment would collapse this to ~5 lines per class. Same pattern in `Search`. Biggest structural cleanup opportunity.~~

**7. [api.py:63](nasa_images/api.py#L63) — `urllib.parse.quote(urllib.parse.unquote(self.nasa_id))` is an idempotency dance.** It re-encodes already-encoded input. For NASA IDs, a plain `quote` would suffice and be clearer. The unquote-then-quote is only meaningful if callers sometimes pass pre-encoded strings, which isn't documented.

**14. [fetch.py:44-49](nasa_images/fetch.py#L44-L49) — flow is slightly awkward.** `meta` is typed `dict | None`, set from `_load_metadata_json`, then reset to `{}` if None. Could just `meta = _load_metadata_json(metadata_url) or {}` and drop the None type.

**15. [fetch.py:59-63](nasa_images/fetch.py#L59-L63) — metadata write is skipped if `meta` is falsy (empty dict from the fallback above).** Correct behavior but subtle — a comment or restructuring would help.

**16.
